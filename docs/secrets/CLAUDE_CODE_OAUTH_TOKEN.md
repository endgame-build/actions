# CLAUDE_CODE_OAUTH_TOKEN

## What it is

A long-lived OAuth token that lets GitHub Actions run Claude Code using your Claude Team or Pro subscription. Generated by the `claude setup-token` command.

## Why it's needed

The changelog pipeline uses `anthropics/claude-code-action@v1` to run 3 AI agents that analyze each PR and synthesize a polished changelog entry. Without this token, the agents fail and the pipeline falls back to raw git-cliff output (unrefined, noisy).

## Where to set it

| Scope | When |
|-------|------|
| **Per-repo secret** | Set on each repo that uses the pipeline |
| **Org-level secret** (preferred) | Set once, available to all repos. Requires org admin. |

## How to create

```bash
# Requires an active Claude Team or Pro subscription
claude setup-token
```

This opens a browser for OAuth consent, then prints the token. Copy it.

If `claude` is not installed:

```bash
# macOS
brew install claude

# Or via npm
npm install -g @anthropic-ai/claude-code
```

## How to set

**Per-repo (you have write access):**

```bash
# Paste the token when prompted
gh secret set CLAUDE_CODE_OAUTH_TOKEN --repo endgame-build/<repo-name>
```

**Org-level (requires admin):**

```bash
gh secret set CLAUDE_CODE_OAUTH_TOKEN --org endgame-build --visibility all
```

**Description for the GitHub settings page:**

> Claude Code OAuth token for AI changelog agents. From `claude setup-token` (Team/Pro subscription). See endgame-build/actions/docs/secrets/CLAUDE_CODE_OAUTH_TOKEN.md.

**Request message for org admin:**

> We need `CLAUDE_CODE_OAUTH_TOKEN` as an org-level secret so every repo can use the AI changelog pipeline without per-repo setup. Generate it with `claude setup-token` (requires Claude Team/Pro subscription), then: `gh secret set CLAUDE_CODE_OAUTH_TOKEN --org endgame-build --visibility all`. The token authenticates Claude Code in GitHub Actions — it uses the subscription, not separate API billing. Tracked in endgame-build/atelier#149.

## Security

- **Blast radius if leaked:** Attacker can run Claude Code sessions charged to your subscription. No access to repos, secrets, or org settings.
- **Rotation:** Run `claude setup-token` again to generate a new token. Update the secret. The old token is invalidated.
- **Expiry:** Tokens are long-lived but tied to the subscription. If the subscription lapses, the token stops working.

## Troubleshooting

| Symptom | Cause | Fix |
|---------|-------|-----|
| `Unable to get ACTIONS_ID_TOKEN_REQUEST_URL` | Missing `id-token: write` permission | Add `permissions: id-token: write` to the calling job |
| `Reached maximum number of turns` | `--max-turns` too low | Increase in `claude_args` |
| Agents succeed but synthesis fails | Synthesis needs more turns than analysis | Set `--max-turns 3` or higher for synthesis |
| All agents fail silently | Token expired or subscription lapsed | Re-run `claude setup-token` and update the secret |
